# Ripley — History

## Learnings

- **2026-05-08:** Joined as Lead for Verdecora Simple project. Goal: simplify enterprise-grade private networking infrastructure to public endpoints. User is Kiko de Angel. Original complex repo at frdeange/verdecoraTest for reference. Working in frdeange/VerdecoraSimpleTest. Target RG: rg-verdecora-simple in Sweden Central.
- **2026-05-08:** Completed the simplification analysis. Main finding: the application architecture is already fit for purpose, but the hosting model is overbuilt around private networking, Front Door, and self-hosted ACA runners. The safest simplification path is public endpoints + managed identity/RBAC + GitHub OIDC, with upload-web kept on direct ACA ingress and Blob SAS uploads retained.
- **2026-05-09:** Completed full functional gap analysis. Main finding: the product intent is solid, but the core E2E is not connected. Biggest blockers are (1) upload-web publishes to a topic while actual workers consume queues, (2) Cosmos database/container config is inconsistent (`albaranes-db` in infra vs `verdecora/processing-records` defaults in services), (3) upload sessions and “Mis albaranes” are still in-memory instead of Cosmos-backed, and (4) HITL and BC posting paths exist as pieces but are not wired into one proven end-to-end business flow.
- **2026-05-09:** Deep pipeline/MCP analysis complete. Written to `.squad/decisions/inbox/ripley-pipeline-mcp-analysis.md`. Core finding: **all 5 MCP servers exist and agent pipeline is structurally complete, but agents run with ZERO tools** because `OrchestratorService` never passes `mcp_tools` to `AlbaranPipeline`. The BC MCP client (`bc_mcp/client.py`) has all 10 operations but no tool adapter bridges it to the agent framework. The ACS email server works but `CommunicationAgentService.handle_hitl_review()` is never called from the orchestrator. The HITL webform publishes decisions to a topic but no consumer resumes the pipeline. Priority fix order: P0 = wire BC MCP tools into pipeline (new `bc_mcp/tools.py` + edit `orchestration.py`), P1 = send HITL email notifications, P2 = HITL decision consumer, P3 = env var normalization.
